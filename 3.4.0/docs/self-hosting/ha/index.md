# OpenVidu High Availability installation

OpenVidu High Availability is part of the PRO edition of OpenVidu. You have the following deployment options:

- [**On-premises installation (DNS Load Balancing)**](https://openvidu.io/3.4.0/docs/self-hosting/ha/on-premises/install-dlb/index.md): Set up OpenVidu High Availability on your own servers with a DNS Load Balancing mechanism.
- [**On-premises installation (Network Load Balancer)**](https://openvidu.io/3.4.0/docs/self-hosting/ha/on-premises/install-nlb/index.md): Set up OpenVidu High Availability on your own servers with a Network Load Balancer.
- [**AWS installation**](https://openvidu.io/3.4.0/docs/self-hosting/ha/aws/install/index.md): Deploy OpenVidu High availability on Amazon Web Services.
- [**Azure installation**](https://openvidu.io/3.4.0/docs/self-hosting/ha/azure/install/index.md): Deploy OpenVidu High Availability on Microsoft Azure.

Once your deployment is complete, refer to the following sections for configuration and management:

- [**On-premises: configuration and administration**](https://openvidu.io/3.4.0/docs/self-hosting/ha/on-premises/admin/index.md)
- [**AWS: configuration and administration**](https://openvidu.io/3.4.0/docs/self-hosting/ha/aws/admin/index.md)
- [**Azure: configuration and administration**](https://openvidu.io/3.4.0/docs/self-hosting/ha/azure/admin/index.md)

If you want to upgrade your OpenVidu High Availability installation, refer to this section:

- [**Upgrade OpenVidu High Availability On-premises**](https://openvidu.io/3.4.0/docs/self-hosting/ha/on-premises/upgrade/index.md)
- [**Upgrade OpenVidu High Availability AWS**](https://openvidu.io/3.4.0/docs/self-hosting/ha/aws/upgrade/index.md)
- [**Upgrade OpenVidu High Availability Azure**](https://openvidu.io/3.4.0/docs/self-hosting/ha/azure/upgrade/index.md)
