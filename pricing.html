OpenVidu Pro is deployed as a cluster following a Master-Slave model (take a look to its architecture). Every minute is charged based on how many CPU cores are available for your cluster(s). OpenVidu Pro is priced as:
$0.0006
per core per minute available for your OpenVidu Pro cluster
Taking into account the following information:

  1. You only pay for your OpenVidu Pro cluster(s) for the time it is running. Usage will be registered the moment you start your cluster and will stop as soon as you shut your cluster down.
  2. You pay for every available core at any given time: if you cluster grows for one hour, that hour you will pay more. If your cluster decreases the next hour, next hour will be cheaper. Currently all Nodes have the same core per minute price.
  3. Your cluster (specifically your OpenVidu Server Pro Node) needs Internet access at all times. If your specific use case doesn't allow your cluster to have Internet access, please contact us through Commercial page.
This price only includes OpenVidu Pro license. If you are deploying your OpenVidu Pro cluster in a cloud provider, you must of course take into account the cost of the cloud resources you will be consuming (basically servers and bandwidth)

OpenVidu Pro usage examples


As explained above, every minute of an OpenVidu Pro cluster is charged according to the number of cores available for the cluster. So let's see some prices of different OpenVidu Pro clusters according to their running time.

Cluster with 6 cores in total

Let's suppose we have our OpenVidu Server Pro Node running in a 2-core server and a single Media Node running in a 4-core server. This topology should be enough for many use cases that do not expect tons of media streams established at the same time.

8 hours of use would cost:
8 hours * 60 minutes/hour * 6 cores * $0.0006 = $1.73

24 hours (1 full day of continuous use) would cost:
24 hours * 60 minutes/hour * 6 cores * $0.0006 = $5.18

720 hours (1 full month of continuous use) would cost:
720 hours * 60 minutes/hour * 6 cores * $0.0006 = $155.52

These prices suppose a continuous usage of the cluster, but rememeber that you can shut it down whenever you are not using it and that you can drop Media Nodes to save resources.

Cluster with 10 cores in total

If we have our OpenVidu Server Pro Node running in a 2-core server and 2 Media Nodes running in 4-core servers, prices will be the following ones:

8 hours of use would cost:
8 hours * 60 minutes/hour * 10 cores * $0.0006 = $2.88

24 hours (1 full day of continuous use) would cost:
24 hours * 60 minutes/hour * 10 cores * $0.0006 = $8.64

720 hours (1 full month of continuous use) would cost:
720 hours * 60 minutes/hour * 10 cores * $0.0006 = $259.20

These prices suppose a continuous usage of the cluster, but rememeber that you can shut it down whenever you are not using it and that you can drop Media Nodes to save resources.

Cluster with 18 cores in total

Let's suppose we have our OpenVidu Server Pro Node running in a 2-core server and 2 Media Nodes running in 8-core servers. This cluster is pretty powerful and should be able to handle a considerable amount of users.

8 hours of use would cost:
8 hours * 60 minutes/hour * 18 cores * $0.0006 = $5.18

24 hours (1 full day of continuous use) would cost:
24 hours * 60 minutes/hour * 18 cores * $0.0006 = $15.55

720 hours (1 full month of continuous use) would cost:
720 hours * 60 minutes/hour * 18 cores * $0.0006 = $466.56

These prices suppose a continuous usage of the cluster, but rememeber that you can shut it down whenever you are not using it and that you can drop Media Nodes to save resources.

Cluster with a variable number of cores

This use case takes advantage of OpenVidu Pro scalablity features. Let's suppose we deploy a cluster with the OpenVidu Server Pro Node running in a 2-core server and a single Media Node running in a 4-core server.

We are fine running this cluster configuration during 8 hours. But suddenly we see a peak in the demand, so we add 2 new Media Nodes to the cluster in 8-core servers to attend the demand (see Scalability).

This high usage level lasts for 8 hours, and then we are fine reducing the cluster size to its initial state. So the last 8 hours of the day we would be running again just a single Media Node in a 4-core server. The price for the full day would be:

First 8 hours wit low demand:
8 hours * 60 minutes/hour * 6 cores * $0.0006 = $1.73

Next 8 hours with high demand:
8 hours * 60 minutes/hour * 22 cores * $0.0006 = $6.33

Last 8 hours with low demand:
8 hours * 60 minutes/hour * 6 cores * $0.0006 = $1.73

Adding up to a total of $9.79 at the end of the day.

How many users can my OpenVidu Pro cluster handle?


This is probably one of the most important questions when using OpenVidu Pro. The number of Media Nodes you need and the size of each Media Node depends on the answer. Therefore, the price of your OpenVidu Pro cluster also depends on the answer.

That being said, there is no single answer to this question. The load each Media Node can handle depends on many factors:

  • The topology of each OpenVidu Session (1:1, 1:N, N:M)
  • The type of media streams being published to the Session (only audio, only video, audio + video)
  • Whether your Sessions are using advanced features such as recording or audio/video filters
You will need to perform some tests for your specific use case, and adapt the size of your cluster accordingly. OpenVidu team can perform these tests for you as part of their commercial services (contact us through Commercial page to ask for an estimate).

How do I get charged when using OpenVidu Pro?


Users must create an OpenVidu account, which is completely free and only requires an e-mail address. Then you will have the possibility of purchasing an OpenVidu Pro license key that can be used to launch your OpenVidu Pro cluster(s).

When purchasing an OpenVidu Pro license key, you will have to indicate your billing address and a credit card.
You will receive a 15-day free trial period during which you will not be charged at all.

After the free trial period, a monthly billing cycle will charge all your expenses to your credit card. Therefore, you will receive an invoice each month. You can review your upcoming expenses and your past invoices in your OpenVidu account page. And don't worry: we don't store any credit card data. The entire billing process is securely done via Stripe.

OpenVidu Pro clusters will automatically report their usage on a recurring basis. That's why they need Internet access at all times. You won't be able to run your OpenVidu Pro cluster unless your OpenVidu Server Pro Node has Internet access (see OpenVidu Pro architecture). If your specific use case doesn't allow your cluster to have Internet access, please contact us through Commercial page.


Pricing FAQs


1. If I don't have any ongoing video session in my OpenVidu Pro cluster, do I have to pay?

Yes, OpenVidu Pro license is charged for any running OpenVidu Pro cluster, even when there are no active sessions. If your cluster is on, you will at least have to pay for the OpenVidu Server Pro Node. But remember you can drop Media Nodes to reduce the cluster size and reduce your bill according to user load. And as a definite measure you can always turn off the cluster so you don't even pay for the OpenVidu Server Pro Node.

2. What is the price per minute of each call?

The cost per minute of video call depends very much on the utilization ratio of your OpenVidu Pro cluster. That is, the number of calls you are able to maintain at the same time. If you use a cluster for an average of only 2 calls per hour, the cost per call will be rather high. If you manage to keep a number of calls adjusted to the size of the cluster, the cost will be lower.

3. If I deploy an OpenVidu Pro cluster in a cloud provider, does its size automatically adapt to the user load?

Yes! You can adapt its size manually from OpenVidu Inspector or by consuming OpenVidu Pro REST API (more info here). Nonetheless automatic elasticity is included in OpenVidu Pro roadmap.

4. If I deploy OpenVidu Pro on premises, can I change the size of my cluster?

Of course. It is important to understand two different concepts when using on premises deployments of OpenVidu Pro: it's one thing the number of physical machines you have available in your cluster, and it's another thing the number of Media Nodes that are actually connected to your cluster.

In OpenVidu Pro the price per minute only counts the Media Nodes actually connected to the cluster. Your OpenVidu Pro license won't be charged for Media Nodes that are disconnected from the cluster even if they are turned on. Media Nodes are always disconnected from the cluster when dropped. If you then want to actually shut down the physical machine, you will have to listen to mediaNodeStatusChanged event until the Media Node reaches terminated status, and then you can safely terminate your server. When deploying OpenVidu Pro in a supported cloud provider, the cluster can automatically handle the launch and termination of the physical machines on its own.

5. Which is the ideal size of an OpenVidu Pro cluster for my specific use case?

As explained here, the answer can vary greatly depending on multiple factors. You can use OpenVidu Load Test tool to perform a load test as similar as possible to your use case. OpenVidu team can perform these tests for you as part of their commercial services (contact us through Commercial page to ask for an estimate).

6. I have built my application using OpenVidu CE. Is it compatible with OpenVidu Pro?

Yes. You don't have to change a thing in your application when upgrading your tier: it will work out-of-the-box with OpenVidu Pro.
